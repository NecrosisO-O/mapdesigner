# MapDesigner 0.1.0 产品功能与特点总览

本文档用于汇总 `MapDesigner 0.1.0` 当前已经具备的功能、产品特点、适用场景与边界，作为后续发布说明、README 收口与版本发布页面文案的参考底稿。

## 1. 项目定位

`MapDesigner` 是一个本地优先的六角格地图设计工具，面向异世界、世界观设定与地理参考创作场景。

`0.1.0` 版本的核心定位是：

- 提供一个可实际使用的本地地图编辑器
- 提供一个可供 AI agent 调用的结构化地图操作接口
- 在 WebUI、CLI 与导出之间保持统一的地图规则

它不是一个在线协作平台，也不是一个偏美术绘制型的专业地图绘图软件。当前版本更强调：

- 规则明确
- 数据可保存
- 操作可回放
- 适合创作参考与自动化修改

## 2. 当前交付形态

`0.1.0` 当前包含三种交付形态：

- 本地 `WebUI`
- 本地 `Fastify` 服务
- 本地结构化 `CLI`

其中：

- `WebUI` 面向人工设计与可视化编辑
- `Server` 负责本地文件读写、导入导出与统一规则服务
- `CLI` 面向脚本调用与 AI agent 调用

## 3. 核心功能总览

`MapDesigner 0.1.0` 已经具备一个完整的基础地图设计闭环：

- 新建地图
- 打开地图
- 编辑六角格单元
- 保存地图
- 另存为新地图
- 导入地图文件
- 导出地图文件
- 导出地图图片
- 使用 CLI 查询地图
- 使用 CLI 对地图执行结构化修改

这意味着当前版本已经可以同时支持两类典型使用方式：

- 设计者在 WebUI 中手工搭建地图
- AI agent 通过 CLI 进行结构化查询、预演、修改与导出

## 4. 地图模型与规则特点

### 4.1 六角格布局

- 地图以六角形单元格展示
- 布局固定为 `flat-top-even-q`
- 原点固定为 `row: 0, col: 0`

### 4.2 坐标与编号

- 对外显示坐标格式为 `R{row}C{col}`
- 内部稳定编号格式为 `cell@{row},{col}`
- 坐标支持负数，便于围绕原点向四周扩展

这套规则同时用于：

- WebUI 展示
- CLI 查询与修改
- 导出与持久化过程中的统一定位

### 4.3 单元格状态模型

当前单元格状态分为：

- `designed`
- `undesigned`

含义是：

- `designed` 代表已正式设计、会写入地图文件
- `undesigned` 代表活动区域中的空白候选格，仅存在于运行时

### 4.4 活动区域扩展规则

地图文件只保存 `designed` 单元格，但运行时会自动生成活动区域：

- 所有 `designed` 单元格
- 它们外围一圈 `undesigned` 单元格

当地图尚未设计任何单元格时，会自动回退到一个 7 格种子区，保证地图总是可开始编辑。

这套规则带来的体验特点是：

- 地图天然可向边缘继续扩展
- 文件不会保存大量无意义空白格
- 清空某格时，外围冗余空白格会被自动回收

## 5. 地形与生态系统

### 5.1 叠加式地图信息表达

每个单元格当前采用“地形 + 生态”叠加表达：

- `terrain` 决定基础地貌
- `biome` 决定生态覆盖

这使地图既能表达地形骨架，也能表达环境氛围。

### 5.2 地形分类

为了提高编辑效率，地形选择不是单层大列表，而是两级结构：

- `Terrain 分类`
- `Terrain`

当前地形分类包括：

- 水域
- 海岸
- 平原与低地
- 高地与山地
- 切割地貌
- 干旱地貌
- 寒冷地貌
- 火山地貌

### 5.3 Terrain / Biome 联动

当前版本已经实现较温和的 terrain / biome 联动筛选：

- 先选地形时，生态列表会缩小到更合理的候选项
- 先选生态时，地形分类与地形列表也会反向缩小
- 如果切换后原值不再兼容，界面会自动清空不兼容项

这套机制的目标不是强硬限制，而是：

- 缩小选择范围
- 减少不必要组合
- 提高编辑舒适度

## 6. WebUI 能力

### 6.1 基础地图管理

WebUI 当前已支持：

- 新建地图
- 打开地图
- 保存地图
- 另存为新地图
- 重命名地图
- 复制地图
- 删除地图

### 6.2 地图编辑能力

当前可对单元格编辑：

- `terrain`
- `biome`
- `tags`
- `note`

并支持：

- 选中单元格后查看与编辑
- 将已设计格清空回到 `undesigned`
- 保存前在表单中暂存修改
- 会话内撤销 / 重做

### 6.3 画布交互

地图画布当前已支持：

- SVG 六角格渲染
- 鼠标选中单元格
- 滚轮缩放
- 拖拽平移
- 以鼠标位置为中心的缩放体验

### 6.4 编辑效率能力

当前 WebUI 已提供基础的效率增强能力：

- 格式刷模式
- 悬停与选中信息展示
- 导出参数折叠面板
- 当前状态提示

### 6.5 导入导出

WebUI 当前支持：

- 导入 JSON 地图
- 导出 PNG

其中 PNG 导出支持：

- 服务端落盘
- 浏览器直接下载

## 7. CLI 与 AI agent 能力

### 7.1 CLI 角色定位

CLI 是 `0.1.0` 的重要组成部分，不只是开发调试工具，而是正式面向脚本和 AI agent 的结构化操作入口。

### 7.2 当前支持的 CLI 命令

当前 CLI 支持：

- `maps list`
- `maps create`
- `maps inspect`
- `maps inspect-cell`
- `maps inspect-area`
- `maps neighbors`
- `maps apply`
- `maps import`
- `maps export-json`
- `maps export-png`
- `maps duplicate`
- `maps delete`

### 7.3 结构化命令修改能力

当前已支持的地图命令包括：

- `set_cell`
- `set_cells`
- `clear_cell`
- `replace_terrain`
- `replace_biome`
- `annotate_cell`

这意味着 AI agent 可以：

- 修改单格
- 批量修改多格
- 清空单格
- 成批替换某类地形
- 成批替换某类生态
- 为单格增加附加标记和备注

### 7.4 查询与预演能力

为了让 AI agent 在落盘前先理解地图并做审慎修改，当前 CLI 已支持：

- `inspect-cell`
- `inspect-area`
- `neighbors`
- `apply --dry-run`

其中 `dry-run` 的特点是：

- 会完整执行命令链
- 会返回完整结果与差异摘要
- 不会写回地图文件

### 7.5 差异回显能力

`maps apply` 当前会返回结构化差异信息，包括：

- `dryRun`
- `map`
- `command_results`
- `changes`

差异项中包含：

- `coord`
- `cell_id`
- `display_coord`
- `before`
- `after`

这让 AI agent 可以在不依赖 WebUI 的情况下判断修改影响。

### 7.6 输出契约特点

CLI 当前统一返回 JSON envelope，适合：

- shell 脚本
- 程序调用
- AI agent 消费

## 8. 渲染与导出特点

### 8.1 渲染方式

当前地图主渲染路径基于纯 SVG：

- WebUI 使用 SVG 场景构建
- 导出也复用同一套渲染逻辑

这带来的优点是：

- Web 显示与导出风格一致
- 易于控制程序化视觉样式
- 便于后续继续扩展导出能力

### 8.2 视觉表达

当前地图视觉表现主要由程序化规则生成，不依赖外部美术资源：

- `terrain` 决定底色
- `biome` 决定叠加纹理
- `status` 决定透明度和边框
- 主 `tag` 可作为角标或重点标记显示

这意味着当前版本更偏：

- 功能型参考图
- 规则型世界设计图

而不是高精度插画风地图。

### 8.3 导出能力

当前已支持导出：

- `JSON`
- `PNG`

PNG 支持两个基础预设：

- `clean`
- `reference`

## 9. 技术架构特点

### 9.1 Monorepo 与分层

项目当前采用 `TypeScript monorepo`，并明确拆分为：

- `apps/web`
- `apps/server`
- `packages/map-core`
- `packages/map-render`

### 9.2 规则复用

最核心的工程特点是：

- WebUI、CLI、Server 共用同一套 `map-core` 规则

这减少了以下风险：

- Web 和 CLI 对同一地图给出不同解释
- 某种命令只在某个入口有效
- 导出结果与编辑视图不一致

### 9.3 本地优先

当前版本不依赖数据库、不依赖云服务，核心数据直接保存在本地文件中：

- 地图主文件位于 `storage/maps`
- 导出文件位于 `storage/exports`

这使它适合：

- 个人创作
- 本地资料库管理
- 离线使用
- 与本地脚本或 agent 工具链集成

## 10. 当前适用场景

`0.1.0` 当前最适合以下场景：

- 异世界地理底图设计
- 小说 / 跑团 / 游戏世界观草图制作
- 以地形和生态为主的世界参考地图
- 需要同时结合人工编辑与 AI agent 自动改图的流程
- 需要本地文件留档与版本管理的创作项目

## 11. 当前版本的主要特点

如果要用简短方式描述 `MapDesigner 0.1.0`，它的主要特点可以概括为：

- 本地优先
- 六角格驱动
- 规则清晰
- 数据结构化
- 既可人工编辑，也可 AI agent 调用
- 支持保存、导入、导出与图片输出

## 12. 当前已知边界与未实现范围

`0.1.0` 仍然是一个初始可用版本，以下能力当前未包含在正式实现范围内：

- 自然语言命令解析
- 账号系统
- 云同步
- 多人协作
- CSV 作为主持久化格式
- 奇幻专用地形扩展
- 人文图层与政治区划层
- 跨会话持久化历史
- 更复杂的批量框选与群组编辑
- 更完整的发布级地图模板系统

因此，这个版本更适合被理解为：

- 一个已经可用的地理参考地图编辑初始版

而不是：

- 一个面向所有地图生产需求的完整终版系统

## 13. 0.1.0 版本发布建议口径

如果将当前版本定为 `0.1.0`，建议采用如下发布口径：

- 该版本重点验证“人工设计 + AI agent 协作改图”的核心闭环
- 后续版本将在批量编辑、人文图层、导出增强与更强查询能力上继续扩展
